爬坑必备

hexo坑

  • 使用git命令行工具可以极大地提高后续操作的便捷度
  • 版本hexo-cli: 1.1.0 node: 8.11.3 这是当前版本
  • 版本很重要
  • 新版：安装便捷，更少bug
  • 旧版：教程更多，更稳定
  • 目前hexo版本问题不大，所以不需要太重视
  • 按照教程问题不大

next史前巨坑

  • 推荐选择next主题

    在没有基础的情况下，next丰富的教程以及全面的功能实在没有拒绝的理由

    • 使用5版本，不要使用6版本

      版本版本版本，坑爹的事情说三遍

      5.14: git clone git@github.com:iissnan/hexo-theme-next.git
      6.0: git clone git@github.com:theme-next/hexo-theme-next.git

    • 如果有强迫症参考版本升级坑教程,next升级教程2，
    • 由于网上大多教程以及当前的官方文档都是基于next5版本的，所以6版本需要爬坑无数—-比如我
    • 不要反复切换教程，看到一个就顺着做下去
    • next官方文档可以看一看，讲的挺全但是不详细
    • next教程推荐：

      教程1
      这个教程讲的十分全面
      教程2
      这个同上
      教程3
      教程4

github page巨坑：

网搜多时发现无数受害者因他走上退路，起始，我也差点卷铺盖走人

  • github仓库名字设置

    githubusername.io 正确
    othername准备卷铺盖走人吧

  • githubpage本身也是一个github仓库

    和普通的代码仓库没有什么区别，所以命令都是通用的，只是以仓库名字的特别性以及仓库种类的特别，提供了静态页面托管功能，按照指定的路径可以访问

  • 配置是在github page一栏 不要 选择主题，还有不要选择disable选项，除非你不想访问并展示博客页面
  • 只是静态网页，不过通过画板娘等配置可以使页面变得很高逼格
  • git操作在最开始需要，到第一次配置完成后，只需要使用hexo clean && hexo g && hexo d实现代码更新部署上传一条龙操作

学习指南

工具

  • git
    • nodejs
    • npm
    • hexo
    • markdown

推荐首先安装git，使用命令行进行操作

选择原因

  • 使用githubpage是为了更长远的发展，可以同时发布在coding上解决百度检索国内访问速度的问题，当然不能翻墙的小伙伴推荐coding
  • 使用hexo轻松搭建美观的个人博客，不需要繁琐的前端代码编辑，只需要一次性折腾的配置+专注写作
  • github官方推荐的Jekyll以及我所使用的hexo都是很棒的工具
  • next目前最火的hexo主题，简约而功能强大，主题可是个坑，这里在下面会详细说明
  • hexo有着丰富的插件，丰富的主题，活跃的社区，丰富的教程，基于nodejs npm等

基本介绍

  • hexo官网教程hexo_doc
  • github page：基于github的代码托管平台，划重点免费，重点：github Pages可以被认为是用户编写的、托管在github上的静态网页，详情githubpage
  • next主题请参考next

  • markdown：please google/baidu

    搭建个人网站思路/流程

  • 思路流程：

    • 先在配置好hexo
    • 本地编写博文
    • 借助hexo的生成网页功能生成html代码
    • 借助hexo上传功能上传到github page
    • 从而生成静态网页
    • next则是为博客选择的主题
      • 你所要做的：配置，写博文，输入hexo s && hexo g && hexo d
      • hexo为你做的：生成html代码，并且通过插件发布到github上
      • next为你做的：通过配置控制生成的html代码，从而控制网页的格式，美化完善个人博客
      • githubpage为你做的：代码托管平台，提供了一个代码呈现以及存储的平台，通过配置域名可以实现更好的推广，当然github.io逼格也蛮高的
